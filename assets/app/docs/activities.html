/activity

API Documentation to Activity entity, an ativity is any taks perform by the user during the day.

Fields

The Activity entity has the fields:

  • id {number} - the activity identifier on database
  • title {string} - the activity name
  • description {string} - the activity description
  • createdAt {date} - the activity creation date
  • creator {number} - Indicating the user owner of this activity

Url Parameters

The server accept the following parameters in the routes

  • limit - The maximum number of results to retrieve
  • skip - The total of resource to skip into database
  • sort - The sort type to apply in the resources (ASC or DESC)
  • creator - The user id to filter results before retrieve activities

4.0 Creating a new Activity

Request parameters (Warning: the creator field is required, the creator parameter represent the User Identifier)


  POST /activity

  //Request data
  {
    title : "Estudar técnicas de compressão de dados",
    description : "Comprimir grande quantidade de imagens de forma eficiente."
    creator : 4
  }

Server response:

{
  "status": 200,
  "activity": {
    "creator": 4,
    "title": "Estudar técnicas de compressão de dados",
    "description": "Comprimir grande quantidade de imagens de forma eficiente.",
    "createdAt": "2016-11-20T16:44:41.057Z",
    "updatedAt": "2016-11-20T16:44:41.057Z",
    "id": 2
  }
}

5.0 Updating an Activity by id

Route do request and data to send.


  PUT /activity/2

  //Request data
  {
    title : "Estudar compressão de dados",
    description : "Comprimir imagens eficientemente."
    creator : 4
  }

Server response:

{
  "status": 200,
  "message": "Activity updated successfully."
}

6.0 Deleting a new Activity

Route do request.

  DELETE /activity/2

Server response:

{
  "status": 200,
  "message": "activity deleted successfully."
}
